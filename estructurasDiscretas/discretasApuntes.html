HLCC(2) RR¶

caracteristicas:¶

1) Tiene a lo más 2 llamadas(orden 2)

2) Solo multiplicar por constantes(cc = coeficientes constantes)

3) Solo sumando(lineal), no hay carculos esperando llamdas recursivas.

4) Teorema : Se utilizan cuando la recursion se da fn = fn-1 + fn-2, cuando suamn tiempos anteriores.

Ejemplo: fibinacci HLCC(2)¶

fn = 1fn-1 + 2 fn-2...... Cumple las restricciones del teorema, porque la constante multiplica al tiempo y al final se suman. No hay nada esperando. HLCC(2) = homogeneo, lineal, coeficiente, constantes, grado 2.

Ejemplos no HLCC(2)¶

1) fn = 2fn/2 + 1

No es HLCC porque de n pasa a n/2, no es lineal

2) fn = nfn-1+2fn-2

n multiplica la llamada recursiva y n no es constante.

1) Resolver polinomio caracteristico

2) Encontrar las raices

1) Mover uno ala vez.

def merge(a, b):
    n, m = len(a), len(b)
    result = [0] * (n+m)
    i, j, k = 0, 0, 0;
    
    while i < n and j < m:
        if a[i]< b[j]:
            result[k] = a[i]
            i += 1
        else:
            result[k] = b[j]
            j += 1
        k += 1
    while i < n:
        result[k] = a[i]
        i += 1
        k += 1
    while j < m:
        result[k] = b[j]
        j += 1
        k += 1
    return result

def quickSort(a:list[int],izquierda, derecha) -> int:
    if izquierda < derecha:
        mitad = particion(a, izquierda, derecha)
        quickSort(a, izquierda, mitad)
        quickSort(a, mitad + 1, derecha)


def particion(a:list[int],izq,derec) -> int:
    pivote = a[izq]
    while True:
        while a[izq] < pivote:
            izq += 1
        while a[derec] > pivote:
            derec -= 1
        if izq >= derec: 
            return derec
        else:
            a[izq], a[derec] = a[derec], a[izq]
        izq  += 1
        derec -= 1

Formalización asintótica: O(), Ω(), Θ()¶

O() f ~ O(g) -> ∃ n0 >= 0, ∃ c > 0 : f(n) <= cg(n) ∀ n >= n0

Una función g multiplicada por una constante en un punto (n0) es >= a f.

Se concluye que f crece a lo más como g basado por una constante

Ω() f ~ Ω(g) -> g ~ O(f)

Una función g multiplicada por una constante g sea < f basado en un punto n0.

Se concluye que f crece al menos como g basado por una constante, f >= cg

Θ() f ~ Θ(g) -> f ~ Ω(g) ^ f ~ O(g)

Una función que al probar los dos casos, tiene una cota O(g) y Ω(g) al mismo tiempo, dandose dos funciones g, con constantes diferentes y diferentes puntos n0.

Teoremas importantes¶

1) c ~ O(1) ∀ c > 0

2) f1 + f2 ~ O( max(f1, f2) )

3) f1 * f2 ~ O( O(f1), O(f2) )

4) si es un polinomio, f ~ O(nd), se usa al monomio más grande

Propiedades Relaciones Binarias:¶

Reflexiva: ∀ x / xRx para todo x se tiene que x esta relacionado con si mismo.

issubclass es reflexiva

Simétrica: ∀ x,y / xRy → yRx, para cuales quiera de dos elementos se tiene que si x se relaciona con y, y tiene que relacionarse con x para ser simétrica.

Transitiva: ∀ x,y,z / xRy ^ yRz → xRz, se tiene que si x se relaciona con y, Y y se relaciona con z, entonces x esta relacionado con z. Donde no necesariamente son elementos diferentes.

Equivalencia: Es reflexiva, simétrica y transitiva.

Irreflexiva : ∀ x / x¬Rx, x no está relacionado consigo mismo.

Antisimétrica: ∀ x,y / ∀ x,y /xRy ^ yRx → x = y, x está relacionado con y, ^ y está relacionado con x, siendo x ^ y el mismo elemento.

Orden Parcial: Reflexiva, antisimétrica y transitiva.

class Relation:
    def __init__(self, A:set, B:set) : # Métodos especiales se denominan Dunder (Constructor).
        self.A = set(A) # Atributo A, se iguala al set que recibe por parámetro.
        self.B = set(B) # Atributo B, se iguala al set que recibe por parámetro.
        self.pairs = set() # Atributo pairs, set de todos los pares de la relación.
    
    def relate(self, x, y) : # Añade un par (x,y) que se encuentra relacionado al set pairs.
        print(f"{x, y} have been related!")
        self.pairs.add((x, y))
    
    def are_related(self, x, y) : # Verifica si un par (x,y) existe en el conjunto de elementos relacionados.
        return (x, y) in self.pairs
    
    def neighbors(self, x) : # Devuelve aquellos pares en los que x está presente.
        return {p[1] for p in self.pairs if p[0] == x}

def is_transitive(R:BinarySelfRelation):
    for x in R.A : # Para cada 'x' en en la relación.
        for y in R.neighbors(x) : # Devuelve cuáles 'y' se relacionan con 'x'.
            for z in R.neighbors(y) : # Devuelve cuáles 'z' se relacionan con 'y'.
                if not R.are_related(x, z) : # Por definición debería existir una relación entre 'x' y 'z' según 'y'.
                    return (x, y, z) # Devuelve un contraejemplo.
    return True

def is_reflexive(R:BinarySelfRelation):
    for x in R.A :
        if not R.are_related(x, x) :# si cada x en A no se relaciona con si mismo, muestra el x
            return(x)
    return True

def is_symmetric(R:BinarySelfRelation):
    for x in R.A :
        for y in R.neighbors(x) : 
            if R.are_related(x, y) and not R.are_related(y, x) :# si un x, y que pertenecen a A, incumple que xRy ^ yRx, muestra x,y
                return (x, y)
    return True

def is_equivalent(R:BinarySelfRelation):
    if is_transitive(R) == True and is_reflexive(R) == True and is_symmetric(R) == True :#si cumple ser reflexiva,simetrica,transitiva da true
        return True
    return False

def translate_to_matrix(R:BinarySelfRelation) -> list : # Construye una matriz según una BinarySelfRelation.
    new_matrix:list = []
    elements:list = list(R.A) # Los elementos siempre serán distintos.

    for i in range (len(elements)) :
        new_matrix.append([0]*len(elements))   
    for i in range (len(elements)) :
        for j in range(len(elements)) :
            if R.are_related(elements[i], elements[j]) :

def warshall_reflexive(R:BinarySelfRelation) :
    warshall_matrix:list = translate_to_matrix(R)
    for i in range(len(R.A)) :
        warshall_matrix[i][i] = 1
    return warshall_matrix
 
def warshall_symmetric(R:BinarySelfRelation) :
    warshall_matrix:list = translate_to_matrix(R)
    for i in range(len(R.A)) :
        for j in range(len(R.A)) :
            if warshall_matrix[i][j] != warshall_matrix[j][i] :
                warshall_matrix[i][j] = 1
                warshall_matrix[j][i] = 1        
    return warshall_matrix
 
def warshall_transitive(R:BinarySelfRelation) :
    warshall_matrix:list = translate_to_matrix(R)
    vertices = len(R.A)
    for k in range(vertices) :
        for i in range(vertices) :
            for j in range(vertices) :
                warshall_matrix[i][j] = warshall_matrix[i][j] or (warshall_matrix[i][k] and warshall_matrix[k][j])
    return warshall_matrix

Grafos¶

Conceptos básicos:

Sea V un conjunto de objetos en un grafo llamados= vértices o nodos.

Sea E un conjunto de (arcos o aristas), usados para unir los nodos de un grafo.

Grafos dirigidos: (x,y) x, y ∈ V (x,y) != (y,x) La arista tiene dirección.

Grafos no dirigidos: {x,y} x,y ∈ V/ {x,y}={y,x} La arista va en ambas direcciones.

Terminologia y notación:¶

Análisis o metódologia de un grafo:

Dado un grafo, se toman en cuenta los siguientes aspectos:

1) Preguntarse si ¿es un dígrafo o un grafo?

2) ¿Es simple o múltigrafo?

3) Si, sí es un múltigrafo, preguntamos ¿Es seudografo?

Grado de un grafo:¶

Grado δ (a) = grado de a, sea a un nodo.

Si es un grafo no dirigido, el grado es igual a la cantidad de arcos, si es un lazo cuenta doble.

Grado de un digrafo:¶

Grado δ - (a) = grado de entrada, sea a un vértice o nodo.

Grado δ + (a) = grado de salida.

Caminos¶

Se toma en cuenta los vértices y arcos por los que se puede pasar para ir de un punto de salida a un punto de llegada.

Cuando es un digrafo no se permite ir y venir por un mismo camino. Cuando es un grafo no dirigido si se puede.

El largo del camino= cantidad de arcos por donde pasa.

Ejemplo:

A a0 B = 1

A a0 B a1 C= 2

Camino simple : no repite arcos.

Camino: puede ser simple o no.

Ciclo del camino: Se inicia y se termina en un mismo vértice.

Grafo conectado:¶

Tiene siempre un camino que lo conecta a otro vértice.

Nota: un grafo puede no estar conectado pero tener dos componenetes conectados, por lo que se dice tiene 2 componenetes conectadas.

Árbol balanceado (AVL )¶

Árbol que esta equilibrado, no esta más cargado de un lado que de otro.

Si la altura de la izquierda - la altura de la derecha es mayor a 1 , el árbol esta desiquilibrado.

La altura tiene que ser 0 o 1 en valor absoluto.

Pasos:

2) Si se desequilibra aplica rotaciónes: simples si son recto, dobles si es sixac

Variables a utilizar:

w = último nodo inserta (hoja)

z = desde w hacia arriba el primero desequilibrado.

y = hijo de z.

x = hijo de y.

SL = simple a la izquierda

SR = simple a la derecha.

Tipos de rotaciónes dobles:

Doble a la izquierda: SL y SR

Doble a la derecha: SR Y SL

Movimientos:

SI la rotación es simple y el árbol esta hacia la derecha = SL y/z

SI la rotación es simple y el árbol esta hacia la izquierda = SR y/z

SI la rotación es doble y el árbol esta en sixac hacia izquierda = 1° SL x/y, 2° SR x/z

SI la rotación es doble y el árbol esta en sixac hacia derecha = 1° SR x/y, 2° SL x/z

class BTree: # Binary Tree
    """
        Class representing a simple Binary tree
        All logic is delegated (bypased) to BNode
        It contains a header (__header) that:
        either
            a) __header is None meaning Tree is empty
            or 
            b) __header is a BNode which is the actual root of the tree
    """
    class BNode: # Binary Node
        """
        Class representing a Binary Node
        Contains minimal logic/structure required for binary  operations
        BNode has data:object (the data in node), left:BNode and right:BNode
        
        """
        #editado por karla Quiros, 8 am, NRC 41711
        def __init__(self, data=None, left=None, right=None):
            self.__data = data
            self.__left = left
            self.__right = right

        @property
        def data(self):
            return self.__data
        @data.setter
        def data(self, data):
            self.__data = data
        @property
        def left(self):
            return self.__left
        @left.setter
        def left(self, node):
            self.__left = node
        @property
        def right(self):
            return self.__right
        @right.setter
        def right(self, node):
            self.__right = node
        def is_leaf(self):
            return not (self.left or self.right)
        @property
        def height(self):
            def h(t):
                return 0 if t is None else t.height
            return 0 if self.is_leaf() else 1 + max(h(self.left), h(self.right))
        
        def __str__(self):
            return str(self.data) if self.is_leaf() else f"{self.data}({self.left}, {self.right})"
        
        def __repr__(self):
            return f"BNode(data={self.data}, left={self.left.__repr__()}, right={self.right.__repr__()})"
